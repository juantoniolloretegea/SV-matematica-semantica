## Función de esta carpeta

La carpeta `registros` conserva los archivos de control criptográfico y temporal asociados al paquete final de preservación. Estos archivos no sustituyen la publicación principal ni los laboratorios reproducibles. Su función es documentar la integridad material del ZIP final mediante hash SHA-256, firma electrónica CMS/PKCS#7 y prueba OpenTimestamps.

El archivo `.ots` corresponde al ZIP final indicado abajo. El archivo `.csig` contiene la firma electrónica del mismo ZIP. Ambos quedan vinculados al mismo paquete por digest SHA-256.

## Interpretación del registro

Este registro certifica la correspondencia material entre el paquete preservado, la prueba OpenTimestamps y la firma electrónica. No certifica una autoridad externa sobre el contenido doctrinal de la publicación; documenta la integridad, trazabilidad y cierre material del lote técnico depositado.

El uso correcto de los identificadores es el siguiente: el DOI PubPub/Crossref identifica la publicación editorial; el DOI de la colección **El Universo** identifica el contexto editorial de colección; el DOI Zenodo identifica el depósito de preservación; el DOI Zenodo de todas las versiones identifica la línea de versiones del registro técnico; GitHub conserva el corpus técnico vivo; Internet Archive conserva capturas históricas.

## Advertencia de consistencia

No debe modificarse este registro, el ZIP final, el `.ots`, el `.csig`, el Markdown principal, los README o los laboratorios sin actualizar de forma coherente todo el conjunto afectado. En particular, cualquier cambio dentro del ZIP final invalida la firma y la prueba OpenTimestamps previas y obliga a generar un nuevo paquete firmado y estampillado.
