# Programa de interfaces del Sistema Vectorial SV

## Finalidad

Esta carpeta agrupa las piezas documentales, experimentales y editoriales relativas al **programa de interfaces del Sistema Vectorial SV**, entendido como el conjunto de carriles destinados a formalizar, probar y publicar la mediación entre el marco doctrinal SV y dominios funcionales diferenciados.

El programa nace con una exigencia metodológica precisa: cada interfaz debe desarrollarse con autonomía suficiente para preservar su propio objeto, pero sin romper la unidad algebraica y semántica del sistema. La vocación de esta carpeta no es puramente archivística. Su función es servir de **raíz operativa y editorial** para publicaciones, datasets, figuras, scripts y metadatos asociados a cada carril.

## Estructura

En el estado actual, la carpeta contiene el carril:

- `semantica/`

Quedan previstos, sin perjuicio de su creación posterior y bajo idéntica disciplina estructural:

- `vision/`
- `motricidad/`

## Criterios de organización

Cada carril deberá tender a una organización homogénea, al menos en los siguientes subespacios:

- `manuscrito/` → textos publicables, versiones de trabajo y testigos de publicación
- `figuras/` → imágenes estructurales, portadas y apoyos visuales del artículo
- `codigo/` → scripts de generación, verificación o apoyo experimental
- `meta/` → manifiestos, descripciones cortas, referencias y documentación auxiliar

Esta homogeneidad no responde a un formalismo externo, sino a la necesidad de que cada interfaz sea:
- rastreable,
- reproducible,
- y editorialmente gobernable.

## Estatuto doctrinal

Las interfaces no sustituyen al núcleo doctrinal del Sistema Vectorial SV. Lo prolongan en planos funcionales concretos. Por ello, ninguna carpeta de este programa debe leerse como repositorio aislado o independiente de la matemática y la semántica del sistema, sino como **despliegue subordinado y estructurado** de sus invariantes.

## Nota de mantenimiento

Si en el futuro se crean nuevos carriles, conviene replicar la estructura aquí usada para `semantica/`, manteniendo continuidad de nombres, README específicos y criterio de trazabilidad documental.
